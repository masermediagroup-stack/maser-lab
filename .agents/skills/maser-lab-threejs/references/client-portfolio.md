# Client and Portfolio Adaptation

Template for reuse notes in every Three.js project `PROJECT.md`.

## Reuse questions

- Could this be a client hero section?
- Could colors swap for another brand?
- Could the object be replaced with a product?
- Could the environment become a branded world?
- Could this become a case study visual?
- Could this be a landing page background?
- Could this simplify for a service business?
- Could this adapt for a marketing campaign?
- Could this become a reusable Maser-Lab component?

## Document per project

### Tyler's personal portfolio

- Section fit (hero, about, case study)
- Narrative tie-in
- Performance budget for portfolio host

### MaserMedia portfolio

- Showcase value
- Brand alignment (Maser blue accent `#10a4ff` from lab tokens)
- Demo chrome compatibility

### Client websites

| Field | Notes |
| --- | --- |
| Reusable | What transfers as-is |
| Brand-specific | What must change per client |
| Tokenizable | Colors, typography, intensity uniforms |
| Content swappable | Copy, logos, product models |
| Visual rules | What must stay (lighting mood, camera angle) |
| Performance limits | Max texture size, mobile fallback |
| Fallback | Required for client production |

### Marketing / landing

- Above-the-fold weight
- CTA legibility over canvas
- SEO: critical content in DOM, not canvas-only
