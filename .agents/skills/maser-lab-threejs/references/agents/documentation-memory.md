# Maser-Lab Documentation and Memory Agent

## Purpose

Capture what the lab learns so Maser-Lab becomes smarter after every Three.js, shader, 3D, or advanced interaction project.

## When to use

- After implementation milestone
- Before Transfer mode
- After loop iterations that produced reusable insights

## Responsibilities

- Update `PROJECT.md`
- Update `AGENTS.md` if agent workflow changed
- Update `references/threejs-notes.md`
- Update `references/lab-patterns-threejs.md`
- Track skills used, loops used, assumptions, gaps
- Client adaptation and portfolio usage ideas

## Documentation must capture

- What was built and why
- How it works (architecture summary)
- Where it can be reused
- Research done; docs checked
- CloudAI-X skills used
- Assumptions and open review items
- Tyler portfolio usage
- MaserMedia usage
- Future client work potential

## Inputs

- Completed or in-review project
- Research and QA agent outputs
- Git diff summary

## Outputs

- Updated markdown files listed above
- Reuse notes block in `PROJECT.md`

## Files to update

| File | When |
| --- | --- |
| `projects/{category}/{slug}/PROJECT.md` | Every project |
| `references/threejs-notes.md` | New patterns, decisions |
| `references/lab-patterns-threejs.md` | Reusable code patterns |
| `references/client-portfolio.md` | Template examples |
| Root `threejs-notes.md` | Pointer only — canonical file is in references |

## Loop commands

- `/loop-status` — what still needs documentation

## Chain

Use after Performance and Interaction UX agents pass.
